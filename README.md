## Project Overview

This project analyzes a dataset of 50,000+ music tracks with audio features (acousticness, energy, danceability, loudness, instrumentalness, valence, tempo, and acoustic_electronic_ratio) to answer three discovery questions:

1. **What acoustic properties naturally group songs?**
2. **Can we explain those groups with simple, interpretable rules?**
3. **Are there unusual tracks (anomalies) that deviate from typical patterns?**

The analysis uses K‑Means clustering, PCA, decision trees, Local Outlier Factor (LOF), and DBSCAN. Key findings:
- **Four interpretable clusters:** acoustic/calm, energetic/danceable, extreme acoustic bias, and balanced/mainstream.
- **Acousticness and energy** are the most discriminating features.
- **5% of tracks** are anomalies, mostly in the extreme acoustic cluster.

## How to Reproduce the Analysis

### 1. Set up the environment

Make sure you have Python 3.8+ installed. Then install the required libraries:

```bash
pip install -r requirements.txt
```

### 2. Generate the dataset

Run the data generation script:

```bash
python generate_music_data.py
```

This creates `music_data.csv` with:
- 50,000 unique tracks
- Multi‑genre duplicates (same track ID, different genre labels) – ~8,000‑12,000 extra rows
- Extreme outlier in `acoustic_electronic_ratio` = **87.4** (as per professor feedback)

### 3. Run the analysis notebook

Launch Jupyter:

```bash
jupyter notebook
```

Open `M4_Final_Analysis.ipynb` and run all cells from top to bottom.

The notebook performs:
- Log‑transform of the ratio to fix skew
- Deduplication experiment (removes exact duplicates)
- K‑Means clustering (k=2..8, final k=4) with silhouette plot
- PCA (variance explained reported)
- Decision tree (depth 4) for interpretable rules
- Anomaly detection using Local Outlier Factor (LOF)
- DBSCAN comparison with k‑distance graph optimisation
- Cluster profiling and interpretation

All tables, figures, and diagnostic prints will appear inline.

### 4. Review the final report

Open `M4_Final_Report.pdf` for a polished 5‑page document covering:

- Methods (preprocessing, techniques, choices)
- Results (clusters, PCA, decision tree, anomalies, DBSCAN)

## Key Results Summary

| Cluster | Description | Size (%) | Key Features |
|---------|-------------|----------|---------------|
| 0 | Acoustic & calm | 12.4% | high acousticness (>0.7), low energy (<0.3) |
| 1 | Energetic & danceable | 41.0% | high energy (>0.7), high danceability (>0.6) |
| 2 | Extreme acoustic bias | 8.0% | very high acoustic_electronic_ratio_log (>2.5) |
| 3 | Balanced / mainstream | 38.6% | medium values across all features |

- **PCA variance captured:** 36.24% (PC1 = 23.63%, PC2 = 12.61%)
- **Silhouette score (k=4):** 0.086 (lower than k=2 but justified by finer subgroups)
- **Anomalies:** 5.0% of tracks, 78% of these belong to Cluster 2
- **DBSCAN vs K‑Means Adjusted Rand Index:** 0.31 (moderate agreement, indicating non‑spherical shapes)

## Critical Assessment (Summary)

### Validity
- K‑Means was run with 10 random seeds; cluster assignments were stable (Rand index >0.95 across seeds).
- Log‑transformation of the ratio was essential – without it, the 87.4 outlier dominated distances.
- DBSCAN parameters were chosen objectively using a k‑distance graph.

### Limitations
- **Data:** Synthetic – may not generalise to real streaming platforms.
- **Method:** K‑Means assumes spherical clusters; DBSCAN’s ARI = 0.31 suggests some non‑spherical structure.
- **PCA:** First two components capture only 36.24% of variance; 63.76% is discarded.

### Ethical Considerations
- **Privacy:** No personal data – tracks are public metadata.
- **Bias:** Broad genre labels could oversimplify; avoid reinforcing stereotypes.
- **Misuse:** Clustering could create filter bubbles; a responsible system would include serendipity.
